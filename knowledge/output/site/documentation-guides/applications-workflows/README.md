# Applications & Workflows

Guides for developing metadata-driven Applications that submit to multiple compute backends and how to write/author apps for the platform.

## Source Documents

- **Developing Metadata-Driven Applications in Isabl** — Introduces Isabl Applications, which let you deploy data science tools across many experiments using metadata-driven rules. It highlights that applications are tool-agnostic, can submit jobs to multiple compute environments (local, cluster, cloud), and store results as tracked analyses.
  [Source](https://docs.isabl.io/writing-applications)
- **Isabl CLI: Commands, Retrieval, and Management** — Summarizes the major capabilities of the isabl-cli package, including creating/executing analyses, retrieving data, and importing/managing files. The page highlights that the CLI exposes many useful functions for common workflows and points users to specific command areas for analyses, data retrieval, and imports.
  [Source](https://docs.isabl.io/other-cli-commands)
